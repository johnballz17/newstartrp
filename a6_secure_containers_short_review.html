What It Does

This mod adds a secure container item that attaches to a player's SecureContainer slot and keeps that container tied to the player's identity.

Its main purpose is to let players keep protected storage through death and reconnects.

How It Works

When the container is attached, the mod saves the owner's player ID on the item. That owner ID is also saved and loaded with persistence.

On player select, the server looks through all secure containers, finds the one owned by that player, and reattaches it to the SecureContainer slot. If none exists, the mod can spawn a starter container instead.

On death, the container is dropped, ownership is kept, lifetime is refreshed, and it is moved away until the owner is restored.

Key Features

  • Owner-based secure container system
  • Persistence through save/load
  • Automatic restore on player select
  • Optional starter container for new players
  • Container damage blocked
  • Nested secure containers blocked
  • Item allow/block list from config
  • Lifetime refresh on detach and death
  • Client receives config from server by RPC
